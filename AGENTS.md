# AGENTS
- must 为必须遵守的规则,绝对不可有任何违背!!
- 全局必须使用UTF-8编码,禁止使用任何非UTF-8编码的文件或内容
- 绝不能用 Get-Content / Set-Content（除非明确加 -Encoding UTF8，但那会写入带 BOM 的文件）。

## 项目关键指令
|     id     |   alias    |          illustrate           |
|:----------:|:----------:|:-----------------------------:|
|   [MUST]   |    必须    |       必须遵守,不可违背       |
| [MUSTNOT]  |    禁止    |      禁止的行为.绝不允许      |
|   [ONLY]   |   只允许   | 只允许使用XXXX 不可使用其他的 |
| [OPTIONAL] |    可选    |  可选择使用XXXX,但不是必须的  |
|   [DOCS]   | 文档锚链接 |        模块上下文文档         |
|   [CODE]   | 代码锚链接 |         模块实际代码          |

## instructions 指令动作

|    instructions    |                             must                             |
| :----------------: | :----------------------------------------------------------: |
|      测试xxx       | 必须实际真实测试,不能凭空想象,必须实质性的全部跑一遍,并发现其中的问题/bug/是否兼容/等系列问题,并保存为md文件,作为交付物的一部分 |
| 创建技能/创建skill |            必须上下文要清晰,必须知道该技能是功能, 作用,并使用.github/skills/AGENTS.md 及  .github/skills/skills-skills/SKILL.md 来创建技能         |
| 重构/优化xxx | 必须先阅读现有代码并理解其职责边界,仅修改必要部分,不得引入无关改动；重构后必须保证原有测试全部通过,或补充对应测试 |
| 写测试/补测试 | 必须覆盖正常路径、边界条件和异常路径；测试命名需体现被测场景；禁止删除或降级现有测试以使新代码通过 |
| 排查CI失败/调试错误 | 必须先查看完整错误日志,定位根本原因后再修改；禁止在未复现问题的情况下盲目修改代码；修复后需确认CI全部通过 |
| 更新/写文档 | 必须与代码实际行为保持一致,禁止描述未实现的功能；文档语言与项目保持一致（中文）；涉及接口变更时同步更新相关示例 |
